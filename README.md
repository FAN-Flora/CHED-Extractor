# 中国三千年疫灾史料智能解析工具

## 📖 项目简介

本项目是一个基于 **DeepSeek API** 的智能分析工具，专门用于从古籍史料（Microsoft Word 文档）中自动提取疫灾事件的六大核心要素。它能够智能识别文档结构（如黑体四号居中的年份标题），将每个段落归属到正确的公历年份，并利用大语言模型精准提取 **疫时、疫域、疫因、疫果、疫种、疫策** 等信息，最终生成结构化的 Excel 表格，方便后续的学术研究和数据分析。

## ✨ 功能特点

- **智能文档结构解析**：自动识别 Word 文档中居中对齐的年份标题（如“崇祯七年（AD1634）”），并将后续正文段落归属到对应年份，解决年份推断难题。
- **多字段精准提取**：输出包含 **15 列** 标准字段（年份、季节、农历月份、原文行政区划记录、历史与现代区划、疫因、疫果、疫种、疫策等）。
- **保留原文表述**：严格遵循“不得翻译”原则，直接摘录原文关键词或完整句子，确保数据忠实于史料。
- **并行处理**：利用 `ThreadPoolExecutor` 并发调用 AI 接口，大幅提升处理效率。
- **可扩展性**：代码模块化设计，易于修改提示词、调整字段或接入其他 AI 模型。

## 🚀 快速开始

### 1. 环境要求

- Python 3.8 或更高版本
- 已安装以下 Python 库：
  - `python-docx`：读取 Word 文档
  - `openai`：调用 DeepSeek API
  - `pandas`：处理 Excel 输出
  - `tqdm`：显示进度条

### 2. 安装依赖

```bash
pip install python-docx openai pandas tqdm
```

### 3. 获取 API Key

本项目使用 DeepSeek 的 API 服务。请前往 [DeepSeek 官网](https://www.deepseek.com/) 注册并获取 API Key。

### 4. 配置

打开 `main.py`（或你的脚本文件名），修改以下配置：

```python
DEEPSEEK_API_KEY = "your-api-key-here"                # 替换为你的 API Key
INPUT_DOCX = r"F:\path\to\your\document.docx"        # 输入 Word 文档路径
OUTPUT_EXCEL = r"F:\path\to\output\result.xlsx"      # 输出 Excel 文件路径
```

也可以通过环境变量设置 API Key，避免在代码中硬编码：

```bash
export DEEPSEEK_API_KEY="your-api-key-here"
```

### 5. 运行

```bash
python main.py
```

程序将自动解析文档，并在控制台显示进度。最终生成的 Excel 文件将保存在指定路径。

## 📂 输入文档格式要求

- **年份标题**：必须为居中对齐，且包含类似 `(AD1634)` 或 `(1634年)` 的格式。示例：`崇祯七年（AD1634）`
- **正文段落**：每个段落通常对应一次疫灾记录（可包含多个句子）。段落长度应大于 10 个字符，且不被视为脚注（如 `[1]: ...`）。

## 📊 输出字段说明

| 列名 | 说明 |
|------|------|
| 代码 | 自增序号 |
| 年份 | 公历年份（从标题解析得到，或由 AI 推断） |
| 季节 | 春、夏、秋、冬等，按原文填写 |
| 月份（农历） | 如“五月”“十月”，按原文填写 |
| 原文行政区划记录 | 史料中的原始地名（如“陕西西安府”） |
| 一级区划 | 历史高层政区（如州、道、路） |
| 二级区划 | 历史中层政区（如府、郡） |
| 县 | 历史县级单位 |
| 省份 | 对应现代省级行政区 |
| 县名 | 对应现代县级行政区 |
| 疫种 | 疫病种类关键词（多个用顿号分隔） |
| 疫因 | 导致疫灾的原因关键词（多个用顿号分隔） |
| 疫果 | 包含疫灾后果的完整原始句子 |
| 疫策 | 包含应对措施的完整原始句子 |
| 备注 | 可填写推断依据、歧义说明等 |

## 🛠️ 自定义修改

### 调整年份识别规则

如需修改年份标题的匹配模式，可编辑 `parse_document_with_years` 方法中的正则表达式：

```python
year_pattern = re.compile(r'[（(](?:AD)?(\d{4})(?:年)?[）)]')   # 匹配 (AD1234) 或 (1234年)
```

### 修改系统提示词

`SYSTEM_PROMPT` 变量中包含了完整的 AI 指令。你可以根据需求修改字段规则、关键词库或示例，以优化提取效果。

### 更换 AI 模型

将 `call_ai` 方法中的 `model` 参数改为其他支持的模型（如 `"deepseek-chat"`），或更换 API base URL。

## 📝 示例

**输入段落**（位于“崇祯七年（AD1634）”标题之后）：

```
陕西西安府大旱，饥，人相食，饿莩盈野，瘟疫大作。
```

**输出行**（部分字段）：

| 年份 | 季节 | 原文行政区划记录 | 疫因 | 疫果 |
|------|------|------------------|------|------|
| 1634 |      | 陕西西安府       | 旱、饥、人相食 |      |

**备注**：AI 根据“人相食”在“瘟疫大作”之前，将其判断为疫因，而非疫果。

## ⚠️ 注意事项

- 确保 API Key 有效，并注意调用频率限制（免费额度有限）。
- 输入文档应避免包含大量无关的脚注或短文本，否则可能被误判为有效段落。
- 若文档中部分年份标题未被识别，请检查其格式是否为居中对齐且包含指定年份模式。
- 生成的 Excel 文件可能与原文档顺序一致，但 AI 解析可能存在个别错误，建议人工抽检。
